# 개념 정리

## 핵심 웹 지표

- 실제 사용 데이터에 따른 페이지의 성능을 보여주는 지표이다. LCP, FID, CLS 세 가지 항목으로 구성되어있다.

## LCP(Largest Contentful Paint)

- 사용자가 URL을 요청한 시점부터 페이지 내에서 시각적으로 가장 큰 콘텐츠를 그리는데에 걸리는 시간
- 사용자는 콘텐츠가 보여지면서 페이지가 실제로 로드되고 있음을 인지하기 때문에 빠르게 그려지는 것이 중요
- 좋음(2.5초 이하), 개선 필요(4초 이하), 느림(4초 초과)

## CLS(Cumulative Layout Shift)

- 페이지 전체 수명 동안 발생하는 모든 Unexpected Layout Shift 중, 가장 큰 Layout Shift의 점수
- 좋음(0.1초 이하), 개선 필요(0.25초 이하), 느림(0.25초 초과)

## FID(First Input Delay)

- 유저와 페이지가 처음으로 상호작용하는 순간부터 (예를 들어, 링크를 클릭하거나, 버튼을 탭하는 등) 브라우저가 해당 인터렉션에 대한 응답으로 실제로 이벤트 핸들러 처리를 시작하기까지의 시간을 측정
- 좋음(0.1초 이하), 개선 필요(0.3초 이하), 느림(0.3초 초과)
