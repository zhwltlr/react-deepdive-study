## 웹페이지의 성능을 측정하는 다양한 방법

### 웹페이지의 성능 측정 방법

- 웹페이지의 성능을 측정하는 방법은 다양합니다.
- 대표적인 성능 측정 도구로는 크롬 개발자 도구, Lighthouse, PageSpeed Insights 등이 있습니다.
- reportWebvitals 함수를 사용하면 create-react-app 프로젝트에서 웹 페이지의 성능을 측정할 수 있습니다.
- reportWebvitals 함수는 Core Web Vitals 지표인 LCP, FID, CLS를 측정하여 성능을 평가합니다.
- NextWebVitals 컴포넌트를 사용하면 Next.js 프로젝트에서 웹 페이지의 성능을 측정할 수 있습니다.

### 구글 라이트하우스

- 탐색모드: 라이트하우스는 탐색모드를 사용하여 웹 페이지의 성능을 측정합니다.
    - 페이지에 접속 했을때 부터 페이지 로딩이 완료될 때까지의 성능을 측정하는 모드다.
    - 지표수집이 완료되면 다음와같은 리포트가 생성된다

    ![state](/assets/seokhyun/kmonstar.png)

    - fcp, lcp, cls외에도 추가적인 지표가있다.
    - time to interactive: 페이지가 사용자와 상호작용할 수 있는 시간
    - speed index: 페이지의 시각적인 완성도를 측정하는 지표
    - total blocking time: 페이지의 대기 시간을 측정하는 지표

- 접근성: 라이트하우스는 웹 페이지의 접근성을 측정하여 접근성 문제를 확인할 수 있습니다.
    - 장애인 및 고령자 등 신체적으로 불편한 사람들이 일반적인 사용자와 동등한 조건에서 웹을 이용할 수 있도록 하는 것을 목표로 한다.
- 권장사항: 라이트하우스는 웹 페이지의 성능을 개선하기 위한 권장사항을 제공합니다.
    - 권장사항을 참고하여 웹 페이지의 성능을 개선할 수 있습니다.
    - 감지된 javascript 라이브러리
    - HTTPS 사용
    - 페이지 로드 시 위치정보 권한 요청 방지하기 등의 권장사항이 있다.

- 트리맵
    - 라이트하우스는 트리맵을 사용하여 웹 페이지의 성능을 시각화합니다.
    - 트리맵을 사용하면 웹 페이지의 성능을 쉽게 이해할 수 있습니다.
    - 트리맵은 웹 페이지의 요소를 계층적으로 표시하여 성능을 시각화합니다.
    - 리소스중 어떠한 파일이 어느정도의 비율을 차지하는지를 확인할 수 있습니다.

    ![state](/assets/seokhyun/treeMap.png)


### WebPageTest

- WebPageTest는 웹 페이지의 성능을 측정하는 온라인 도구입니다.

    ![state](/assets/seokhyun/webPageTest.png)

    - 기본적으로 인도 미국 등 서버의 거리가 다소 먼 곳을 기준으로 테스트를 지원하기때문에 글로벌 웹사이트를 구축하는경우에 좀 더 신경쓸 필요가 있다.
    
### 크롬 개발자 도구

- 크롬 개발자 도구는 웹 페이지의 성능을 측정하는 도구입니다.
- 크롬 개발자 도구의 Performance 탭을 사용하여 웹 페이지의 성능을 측정할 수 있습니다.

     ![state](/assets/seokhyun/chromTest.png)

    - 녹화를 시작하고 웹 페이지를 로드한 후, 녹화를 중지하여 성능을 측정할 수 있습니다.
    - 퍼포먼스에서 메뉴중 의도적으로 스로틀링을 적용하거나 측정 프로필을 저장 및 불러오는 기능들 또한 포함되어있으니 적절히 사용할 필요가 있어보인다.
    - 소요시간과 기본은 성능탭의 핵심이라고 할 수 있다.
    - 시간의 흐름에 따라 메인 스레드의 작업은 어떻게 이루어졌는지, 힙 영역은 어떻게 변화하는지 등을 확인할 수 있다.

### 정리

- 웹페이지의 성능을 개선하고 병목해소등의 작업은 개발하는것 보다도 어려운 작업이다.
- 이를 위해서는 꾸준한 탐색이 필요하며, 이를 위한 도구들을 적절히 활용하는것이 중요하다.
- 0.1초의 개선으로 사용자 경험이 증가하는것을 생각하면서 조금이라도 성능 개선을 할 수 있는 개발능력을 갖춰보도록 해야겠다.
