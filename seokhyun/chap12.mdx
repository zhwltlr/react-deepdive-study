## 모든 웹 개발자가 관심을 가져야 할 핵심 웹 지표

### 핵심 웹 지표란?

- 핵심 웹 지표(Core Web Vitals)는 사용자 경험을 측정하는 지표로, 웹 페이지의 성능을 평가하는 데 도움이 됩니다.
- 핵심 웹 지표는 크게 세 가지로 구분됩니다.

  - LCP(Largest Contentful Paint): 가장 큰 콘텐츠 요소가 화면에 표시되는 시간
  - FID(First Input Delay): 사용자가 페이지와 상호작용할 때의 대기 시간
  - CLS(Cumulative Layout Shift): 페이지의 레이아웃 안정성을 측정하는 지표

- LCP
  - LCP는 페이지의 가장 큰 콘텐츠 요소가 화면에 표시되는 시간을 측정합니다.
  - LCP가 2.5초 이하인 경우 페이지의 성능이 좋다고 판단됩니다.
- FID
  - FID는 사용자가 페이지와 상호작용할 때의 대기 시간을 측정합니다.
  - FID가 100밀리초 이하인 경우 페이지의 성능이 좋다고 판단됩니다.
- CLS
  - CLS는 페이지의 레이아웃 안정성을 측정하는 지표로, 사용자가 의도하지 않은 레이아웃 변화를 경험하는 정도를 측정합니다.
  - CLS가 0.1 미만인 경우 페이지의 성능이 좋다고 판단됩니다.

### 핵심 웹 지표 측정 방법

- 핵심 웹 지표를 측정하기 위해서는 크롬 개발자 도구를 사용할 수 있습니다.
- 크롬 개발자 도구의 Performance 탭에서 녹화를 시작하고 웹 페이지를 로드한 후, 녹화를 중지합니다.
- 녹화된 데이터를 확인하여 LCP, FID, CLS 값을 확인할 수 있습니다.
- Google PageSpeed Insights, Lighthouse 등의 도구를 사용하여 핵심 웹 지표를 측정할 수도 있습니다.

### 핵심 웹 지표 개선 방법

- 핵심 웹 지표를 개선하기 위해서는 다음과 같은 방법을 고려할 수 있습니다.
  - 이미지 최적화: 이미지의 용량을 줄여서 페이지 로딩 속도를 개선합니다.
  - 리소스 압축: CSS, JavaScript, HTML 파일을 압축하여 파일 크기를 줄입니다.
  - 레이지 로딩: 페이지의 이미지나 동영상을 레이지 로딩하여 초기 로딩 속도를 개선합니다.
  - 코드 스플리팅: 페이지의 코드를 여러 번들로 나누어 초기 로딩 속도를 개선합니다.
  - 서버 사이드 렌더링: 서버 사이드 렌더링을 사용하여 초기 로딩 속도를 개선합니다.

### 핵심 웹 지표는 아니지만 성능 확인에 중요한 지표들

- 핵심 웹 지표 외에도 성능 확인에 중요한 지표들이 있습니다.

  - TTFB(Time to First Byte): 서버로부터 첫 번째 바이트를 받는 시간
  - TTI(Time to Interactive): 페이지가 상호작용 가능한 상태가 되는 시간
  - TBT(Total Blocking Time): 페이지가 블로킹되는 시간
  - FCP(First Contentful Paint): 페이지의 첫 번째 콘텐츠가 표시되는 시간
  - SI(Speed Index): 페이지의 로딩 속도를 측정하는 지표

- TTFB
  - TTFB는 서버로부터 첫 번째 바이트를 받는 시간을 측정합니다.
  - TTFB가 200밀리초 이하인 경우 페이지의 성능이 좋다고 판단됩니다.
- TTI
  - TTI는 페이지가 상호작용 가능한 상태가 되는 시간을 측정합니다.
  - TTI가 3초 이하인 경우 페이지의 성능이 좋다고 판단됩니다.
- TBT
  - TBT는 페이지가 블로킹되는 시간을 측정합니다.
  - TBT가 300밀리초 이하인 경우 페이지의 성능이 좋다고 판단됩니다.
- FCP
  - FCP는 페이지의 첫 번째 콘텐츠가 표시되는 시간을 측정합니다.
  - FCP가 2초 이하인 경우 페이지의 성능이 좋다고 판단됩니다.
- SI
  - SI는 페이지의 로딩 속도를 측정하는 지표로, 페이지의 로딩 속도가 빠를수록 값이 낮습니다.
  - SI가 3초 이하인 경우 페이지의 성능이 좋다고 판단됩니다.

### 결론

- 핵심 웹 지표는 사용자 경험을 측정하는 지표로, 웹 페이지의 성능을 평가하는 데 도움이 됩니다.
- 웹페이지의 성능이 좋다고 모든 사용자 경험이 좋아지는것은 아니지만 다수의 사용자들이 웹페이지의 성능에 민감하다는것을 생각했을때 성능개선은 꼭 필요할것으로 생각됩니다.
- 핵심 웹 지표를 개선하기 위해서는 이미지 최적화, 리소스 압축, 레이지 로딩, 코드 스플리팅, 서버 사이드 렌더링 등의 방법을 고려할 수 있습니다.
- 핵심 웹 지표 외에도 TTFB, TTI, TBT, FCP, SI 등의 성능 확인에 중요한 지표들이 있으며, 이러한 지표들을 함께 고려하여 웹 페이지의 성능을 개선할 수 있습니다.
- LCP, FID, CLS 등의 지표를 적절히 확인하며 성능개선에 반영하면 좋을것 같습니다.
